The maintainer of jdx/mise reported two issues with entire-graph:
Two reproducible cases on jdx/mise (Rust workspace, ~3k files). (1) Wrong relationship: which_shim in src/shims.rs gets USES_TYPE/PARAM_TYPE edges to crates/vfox/src/config.rs:struct:Config, but the file explicitly imports crate::config::Config and the correct symbol exists in the graph — 981 of 984 edges targeting the vfox Config come from mise's src/ and are all wrong. CALLS already does import resolution; type/TESTS edges don't, so any common name (Config, Settings, Error) poisons the graph cross-crate. Same root cause produces a wrong TESTS edge (test_parse_array in mise-interactive-config → TomlParser.parse_array in a crate it never touches). (2) Silent miss: commit e3f5ddef2 changed 3 files, but graph commit/graph diff report only src/shims.rs. The src/cli/exec.rs changes (+34/−6 inside two cfg-gated same-name exec_program functions) produce zero output, even though body_changed fires fine for other Rust files in the same range and the snapshot disambiguates the overloads with #sig IDs — the diff reconciler loses them. The changed PowerShell file is also omitted with no "skipped" marker, so empty output can't be distinguished from "not analyzed."
